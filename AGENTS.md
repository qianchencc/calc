# AGENTS.md

本文件约束所有在本仓库工作的自动化代理和维护者。

## 开始前

1. 先阅读 [MAINTENANCE.md](./MAINTENANCE.md)。
2. 把 GitHub `qianchencc/calc` 的 `main` 视为生产源码的唯一事实来源。
3. 检查当前修改范围，保留与任务无关的用户改动。

## 计算规则

- 不得把站内余额称为或当作真实美元。
- 相对官方容量必须同时使用真实人民币/美元汇率、实际到账站内余额和公开计费倍率。
- 阶梯容量必须分段计算，不得把全部余额套用最终倍率。
- 不得在没有一手来源的情况下修改默认汇率。
- 不得在没有可审计样本的情况下修改模型拟合系数。
- 所有数字输入必须保留空字符串编辑态；空值计算为 0，不能自动回填成阻碍覆盖输入的 `0`。
- 修改公式、阶梯、汇率、模型因子或单位时，同步更新界面解释、维护文档和回归案例。

## 修改原则

- 保持中文界面、键盘可操作性、语义标签和移动端可读性。
- 不引入无必要的服务端、数据库、分析脚本或运行时密钥。
- 不把旧 ChatGPT Sites 的平台内部文件、Cloudflare Worker 或服务器部署目录复制进仓库。
- 不提交 Token、密码、`.env*`、用户账单或原始请求日志。
- 依赖升级与业务计算修改分开提交。

## 验证

提交前运行：

```bash
npm install
npm run typecheck
npm run build
```

涉及计算或输入交互时，至少验证：

- ¥100、到账 100、汇率 6.7989、默认阶梯 => 269.44 官方标价容量，约 18.32× 官方直购容量。
- 上述容量选择 Terra => 约 688.43M Token。
- ¥10、到账 10、汇率 10、单倍率 0.4 => 25 官方标价容量，25× 官方直购容量。
- 清空数字框后输入 7 => 结果为 7，不出现 `07`、`0x` 或无法覆盖。
- 30、100、150 三个阶梯边界及移动端高级模式。

## 发布

- 推荐通过分支和 Pull Request 合并到 `main`。
- GitHub Actions 通过后，由 Vercel 自动发布生产版本。
- 发布后验证 [calc.qianc.ltd](https://calc.qianc.ltd) 为 HTTPS 200，并手测受影响路径。
- 紧急 Vercel 回滚后，必须在 GitHub 做对应 revert，不能让线上长期偏离 `main`。

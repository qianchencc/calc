# Domain docs

本仓库采用单一领域上下文。

## 开始工作前

按任务需要读取：

- 根目录 `CONTEXT.md`
- `docs/adr/` 中与任务相关的架构决策

如果这些文件尚不存在，继续工作即可，不应把缺失本身视为阻塞。领域词汇和ADR由 `/domain-modeling` 或 `/grill-with-docs` 在实际形成决定时按需创建。

## 文件布局

```text
/
├── CONTEXT.md
├── docs/
│   ├── adr/
│   └── agents/
└── app/
```

## 词汇规则

Issue、规格、测试和代码必须使用 `CONTEXT.md` 中确定的领域词汇，避免重新引入“站内余额等于美元”等已消除的歧义。

如果需要的概念尚未出现在词汇表中，应重新检查它是否是项目真实概念；确有缺口时，通过 `/domain-modeling` 补充。

## ADR冲突

如果新方案与既有ADR冲突，必须明确指出冲突及重新讨论的原因，不得静默覆盖原决定。

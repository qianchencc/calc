# 使用动态模型拟合控制面

模型注册策略和当前生效拟合会随外部数据变化，继续把它们维护在前端常量中会要求每次更新都重新部署，并诱发模型名称特判。系统采用Upstash Redis保存动态模型注册表、拟合提交、候选、版本水位和生效快照；GitHub仍负责代码、数据结构、初始化种子与灾备基线。注册表是模型行为的唯一入口，收费、拟合、展示和别名分别配置，不在业务代码中针对具体模型分支判断。

## Consequences

运行时模型配置和拟合数据的事实来源将从GitHub常量迁移到Redis，项目因此需要运行时存储、环境密钥、审计与灾备读取。计算器服务端读取生效快照，不提供面向外部的公开拟合读取接口；外部系统只通过受鉴权保护的写入接口提交候选。

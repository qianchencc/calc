# 存储倍率中性的模型产出系数

外部样本可能产生于0.4、0.28或其他计费倍率。若API和存储长期使用某个倍率下的Token系数，前端极易再次重复应用倍率。系统只持久化倍率中性的模型产出系数：总Token（M）除以倍率前计费基数；计算器对每个余额档位使用“档位余额 × 模型产出系数 ÷ 档位倍率”。外部提交必须包含足以让服务端复算该系数的聚合统计量，不能只提交一个不可审计的最终系数。

## Consequences

0.4仅属于历史样本解释和当前阶梯配置，不再是长期数据契约。用户界面仍以“花多少站内余额预计得到多少Token”解释结果，不把内部中性系数包装成用户关心的官方价格。
